# نظرة عامة على نُسُج — لماذا، وماذا، ولمن؟

## الفكرة في جملة واحدة

> **نُسُج يقرأ المدينة كما يقرأ الطبيب قلبَ مريضه: نبضات (IoT)، صور (Remote Sensing)، تخطيطات (BIM)، وقصص المواطنين (Open311). ثمّ يكشف الخلل قبل أن يصبح أزمة.**

---

## المشكلة

التخطيط الحضري السعودي يواجه ثلاث فجوات:

1. **فجوة الرؤية:** البيانات موزَّعة على عشرات الجهات (هيئة الإحصاء، أمانات، مرور، شركات اتصالات...) دون منصة موحَّدة.
2. **فجوة الزمن:** القرار يأتي بعد الأزمة (ازدحام، فيضان، تلوُّث) لا قبلها.
3. **فجوة المواطن:** قرارات تُتَّخذ من فوق دون نبض من تحت.

## القيمة لكل فئة مستفيدة

### للأمانات والبلديات
- **لوحة فابريك (Fabric Dashboard):** خرائط ملوّنة بمؤشرات Eff/Res/Con لكل حي.
- **نقاط ساخنة/باردة (Gi\*):** «أين تتجمَّع المشاكل إحصائيًا؟»
- **محاكاة سيناريوهات:** «ماذا لو نقلنا 10% من حركة المرور إلى الميترو؟».
- **تقارير تلقائية بالعربية والإنجليزية** عبر `/insight/report`.

### للوزارات والقطاع الحكومي
- **موصلات مدمجة** مع: GASTAT (السكان)، Balady (التصاريح)، STAC (الأقمار).
- **التزام بمعايير OGC API Features + STAC 1.0** للتشغيل البيني.
- **هوية وطنية موحَّدة عبر نفاذ** (لاحقًا).

### للمستثمر والمطوِّر العقاري
- **مؤشر التنمية (Growth Hotspots):** «أين فرص التنمية ذات العائد الأعلى؟».
- **خرائط TOD Suitability** — ملاءمة التطوير حول محطات النقل.
- **خرائط 2SFCA** — قياس فُرَص الوصول للخدمات.

### للمواطن
- **تطبيق PWA يعمل دون اتصال.**
- **تسجيل دخول عبر نفاذ.**
- **بلاغات Open311 مع صور وإحداثيات.**
- **إشعارات Push** لمتابعة البلاغ.

## الأعمدة الثمانية للعمارة

| العمود | التقنية | الموقع في المستودع |
|---|---|---|
| **1. الواجهة** | Next.js 14 (App Router) + Leaflet + RTL | `nasij_v3_ui_full/` و كل `apps/web/` |
| **2. API الإدارة** | NestJS 10 + Swagger + JWT | كل `services/api/` |
| **3. ETL والتحليل** | FastAPI + NumPy + Pandas + scikit-learn + Shapely | كل `services/etl/` |
| **4. التخزين** | PostgreSQL 15/16 + PostGIS + pgvector | كل `infra/sql/` و `infra/db/init/` |
| **5. التوجيه** | OSRM (Open Source Routing Machine) | حاوية `osrm` |
| **6. IoT** | MQTT (Mosquitto) + paho-mqtt + JSONB | `services/iot/` |
| **7. الهوية** | Keycloak (OIDC) + Nafath (SSO سعودي) | في v5.5+ |
| **8. المراقبة** | Prometheus + Grafana | `infra/prometheus/` و `infra/grafana/` |

تفاصيل العمارة الكاملة في [`SYSTEM.md`](SYSTEM.md).

## الخصائص الجوهرية التي يجب أن تعرفها

1. **النمط المعماري:** خدمات مصغَّرة متعدِّدة اللغات (TS + Python + SQL).
2. **النموذج المكاني:** خلايا متعدِّدة الأضلاع (`urban_cells`) + هكسات H3 (`urban.h3_cells`).
3. **الترجيح المتعدِّد المعايير:** AHP لاستخراج الأوزان، TOPSIS لترتيب البدائل.
4. **كشف العنقدة المكانيّة:** Getis-Ord Gi* + Moran's I + DBSCAN + Isolation Forest.
5. **الانفتاح المعياري:** OGC API Features + STAC 1.0 + GeoJSON + CQL2 (نص + JSON).
6. **التوسعية بالإضافات:** نظام Plugins بصيغة ZIP عبر سوق `marketplace` في v3.1-pro.

## ما لا يفعله نُسُج (الحدود)

- **ليس نظام GIS عام** مثل QGIS أو ArcGIS — هو منصّة تحليلات لقياسات معيَّنة.
- **ليس بديلًا عن BIM** — لكنّه يستوعب طبقات BIM عبر `bim_gateway` (v3.1-pro).
- **ليس نظام محاسبة أو ERP** — يركّز على الذكاء العمراني.
- **ليس مفتوح المصدر بالكامل بعد** — الترخيص الحالي مغلق (`license: proprietary` في STAC).
