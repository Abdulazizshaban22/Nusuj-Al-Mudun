# nasij_monorepo_v1_2 — التحوُّل الجغرافي + OSRM

> **نقفز من «مكتبة خوارزميّات» إلى «منصّة جغرافيّة فعليّة».**

## الميزات المُضافة على v1.1

### نقاط نهاية جغرافيّة
- `GET /geo/cells` — Feature collection بكل الخلايا.
- `GET /geo/cells_with_indicators` — مع المؤشِّرات.
- `POST /upload/urban_cells` — رفع GeoJSON.

### مؤشِّرات استشعار جديدة
- `POST /rs/evi` — Enhanced Vegetation Index.
- `POST /rs/savi` — Soil-Adjusted VI.
- `POST /rs/ndwi` — Normalized Difference Water Index.
- `POST /rs/change_diff` — تحليل تغيُّر مع إحصاءات.

### Deviation Scoring
- `GET /deviation/score_from_indicators` — `1.0 - (0.6·eff + 0.4·con)`.

### إمكانيّة الوصول (OSRM)
- `POST /access/isochrone` — Isochrone polygon عبر OSRM `/table/v1/driving/`.
- يقطع الطلب إلى دفعات 90 نقطة كحدّ أقصى.
- يستخدم Convex Hull للنقاط القابلة للوصول.

### واجهة جديدة
- `/map` — Leaflet map يحمل `/geo/cells_with_indicators`.
- `/access` — تفاعل بالنقر لاستكشاف الـ Isochrones.

## التغييرات الجوهريّة

### كسر في النموذج المكاني (⚠️ تحذير)
نوع `geom` تحوَّل من:
```sql
geom GEOGRAPHY(POLYGON, 4326)   -- في v1.1: حسابات كرويّة دقيقة
```
إلى:
```sql
geom geometry(Polygon, 4326)    -- في v1.2: حسابات مسطَّحة، أخطاء ~10-15% عند الرياض
```

**التوصية:** أضف عمودًا projected `geom_3857` أو ارجع للنوع GEOGRAPHY.

### Grafana أُزيلت
- في v1.1 كانت Grafana موجودة كحاوية.
- في v1.2 أُزيلت من docker-compose.

### Schema يحوي `DROP TABLE IF EXISTS`
- 🔴 كل docker-compose down يفقد البيانات.
- **التوصية:** تبنّي migration system.

## الـ Dependencies المُضافة على ETL
- `psycopg` — اتصال PostgreSQL.
- `Shapely` — geometry parsing/manipulation.
- `requests` — HTTP client لـ OSRM.

## استخدام مقترَح

تجريبي. مناسب لاستكشاف فكرة Isochrones، لكن لا تنشر إلى إنتاج بسبب:
- DROP TABLE في schema.
- geometry بدلاً من geography.
- لا مصادقة على endpoints.

→ النسخة التالية: [`v2.1`](v2_1.md) تُضيف Keycloak.
