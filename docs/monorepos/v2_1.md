# nasij_monorepo_v2_1_admin_studios — عشرة استديوهات + Keycloak

> **القفزة المفاهيميّة الكبرى: مدينة واحدة، عشر قراءات تخصُّصية.**

## الميزات المُضافة على v1.2

### الاستديوهات العشرة
أوّل ظهور لفكرة «الاستديو». 10 صيغ ترجيح لـ eff/res/con لمجالات مختلفة. تفاصيل في [`../reference/STUDIOS.md`](../reference/STUDIOS.md).

### نقاط نهاية جديدة
- `GET /studio/scores?studio=Mobility` — نقاط الاستديو لكل خلية.
- `GET /sim/run?name=Traffic Shift&delta=0.1` — تشغيل سيناريو (3 منفَّذة من 10).
- `GET /admin/users`، `POST /admin/users`، `PUT /admin/users/:id/role` — إدارة المستخدمين.
- `GET /swagger` — Swagger UI.

### الهويّة والمصادقة
- **Keycloak 26.0.0** على :8081 (development mode).
- **NextAuth + Keycloak provider** في الواجهة.
- Middleware يحمي `/admin`، `/studios`، `/simulations`.

### واجهات جديدة
- `/admin` — إدارة المستخدمين.
- `/studios` — اختيار استديو ورؤية النقاط.
- `/dashboard` — KPIs (Fabric Index, Sprawl Risk, Harmony).
- `/simulations` — اختيار سيناريو.

## ثغرات حرجة

- 🔴 **Admin API بلا حماية** — لا JWT guards (انظر AUDIT SEC-C-03).
- 🔴 **NEXTAUTH_SECRET fallback** = `"please-change"`.
- 🔴 **CORS مفتوح** (`cors: true`).
- 🔴 كلمات مرور افتراضيّة لكلّ شيء.
- 🟠 سيناريوهات 4-10 stubs غير منفَّذة.
