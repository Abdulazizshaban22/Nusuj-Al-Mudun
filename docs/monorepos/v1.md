# nasij_monorepo_v1 — الأساس

> **النسخة الأولى. هيكل عظمي معماري يُثبت الفكرة قبل الميزات. ~5% اكتمال للميزات لكن 100% للعمارة.**

---

## ما يفعله

- يُشغِّل docker-compose كامل: 8 خدمات (DB + API + Web + ETL + Prometheus + Grafana + OSRM + pg_tileserv).
- يقدِّم API هيكليّ على :8088 (GET /health، GET /api/fabric/efficiency stub).
- يقدِّم ETL هيكليّ على :9101 (GET /ingest stub + GET /metrics).
- يقدِّم Web Skeleton مع 6 routes placeholders.
- يحتوي 11 مجلَّد engines/ فارغ.

## ما لا يفعله

- لا توجد بيانات حقيقيّة.
- لا توجد خوارزميّات.
- لا توجد مصادقة.
- لا توجد فهارس.

## المكوِّنات

| المسار | المحتوى |
|---|---|
| `services/api/` | NestJS 10 hello-world |
| `services/etl/` | FastAPI 0.111 stub |
| `apps/web/` | Next.js 14 + 6 صفحات |
| `engines/` | 11 مجلَّد فارغ (وعود) |
| `infra/docker-compose.yml` | 8 خدمات |
| `infra/sql/schema.sql` | `urban_cells` + `fabric_indicators` |
| `infra/prometheus/prometheus.yml` | يحصد ETL فقط |
| `infra/grafana/.../datasource.yml` | يعرض Prometheus |

## كيف تشغِّله

```bash
cd "01- nusoj code/nasij_monorepo_v1/"
cp .env.example .env
docker-compose -f infra/docker-compose.yml up -d
# انتظر ~30 ثانية
curl http://localhost:8088/health           # {ok:true, service:'nasij-api'}
curl http://localhost:9101/ingest            # {ok:true, message:'stub ingestion'}
open http://localhost:3000                    # Next.js home
open http://localhost:3001                    # Grafana (admin/admin)
```

## ثغرات معروفة

- 🔴 كلمات مرور افتراضيّة `nasij` و `admin:admin`.
