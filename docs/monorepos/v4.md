# nasij_v4_monorepo — إعادة الهيكلة (PostgreSQL 16 + pgvector)

> **القفزة إلى PG16 + pgvector + h3_cells + AI embeddings. الإصدار الأكثر طموحًا تقنيًّا.**

## التغييرات الجوهريّة

### Database
- ترقية إلى **PostgreSQL 16** + PostGIS 3.4.
- إضافة **pgvector** للـ AI embeddings.
- إضافة **postgis_raster** لـ الصور.
- Schema جديد `urban.` يحوي:
  - `urban.h3_cells` (BIGSERIAL، h3index UNIQUE، centroid، props JSONB).
  - `urban.citizen_reports` (Open311).
  - `urban.embeddings` (vector(384)).

### Services
- **API:** NestJS placeholder (مستورد لكن لم يُستخدم — Express server).
- **ETL:** FastAPI + الحزمة الكاملة (numpy 2.1, scikit-learn 1.5, pandas 2.2).
- **Web:** Next.js 14 placeholder.
- **MQTT:** Mosquitto **لكن `allow_anonymous true`** (🔴 ثغرة حرجة).
- **Auth:** Keycloak (commented out في docker-compose — لم يُفعَّل بعد).

### Makefile
- `make dev` — docker compose up.
- `make down` — docker compose down -v (مدمِّر).

## ثغرات

- 🔴 MQTT بلا مصادقة (`allow_anonymous true`).
- 🔴 hardcoded creds في .env.example.
- 🔴 لا FK constraints على `urban.embeddings.entity_id`.
- 🔴 NOT NULL مفقود على `citizen_reports.kind` و `geom`.
- 🟠 geom type `geometry(GEOMETRY, 4326)` يقبل union types.
- 🟠 لا فهارس على معظم الجداول الجديدة.

## استخدام مقترَح

نموذج معماري للقفزة التقنيّة. يحتاج تصلية أمنيّة جوهريّة.

→ النسخة التالية: [`v4.7`](v4_7.md) للـ OGC/STAC.
